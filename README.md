# 🕵️ AuditD для Windows

**AuditD** — это легковесная система сбора аудита и мониторинга системных событий для Windows. Вдохновлённая одноимённым демоном Linux (`auditd`), эта утилита позволяет регистрировать критические действия в операционной системе, помогая администраторам выявлять аномалии, отслеживать изменения и соответствовать требованиям безопасности.

> 🚀 **Одним EXE-файлом** — без установки, зависимостей и сложных конфигураций.

---

## 📌 Основные возможности

| Функция                                         | Описание                                                                                               |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| 📋 **Сбор системных событий**                   | Отслеживает запуски процессов, изменения файлов, доступ к реестру и другие значимые действия.          |
| 🗂️ **Логирование в реальном времени**          | События сохраняются в структурированном виде для последующего анализа.                                 |
| 🔐 **Низкое воздействие на производительность** | Оптимизирован для работы в фоновом режиме без заметной нагрузки на CPU.                                |
| ⚙️ **Гибкая настройка**                         | Поведение системы определяется через простые конфигурационные файлы (подробнее в разделе «Настройка»). |
| 🔄 **Автоматический запуск**                    | После активации AuditD самостоятельно обеспечивает свою работу при каждой загрузке системы.            |

## 📥 Установка

1. Скачайте последнюю версию `auditd.exe`.
2. Поместите исполняемый файл в любую папку на целевом компьютере (например, `C:\Tools\AuditD`).
3. **(Рекомендуется)** Добавьте папку в переменную окружения `PATH` для удобного запуска из командной строки.

Никаких дополнительных библиотек или установщиков не требуется.

## 🚀 Использование

### Запуск и активация

1. Запустите командную строку **от имени администратора**.
2. Выполните команду:

```cmd
auditd.exe --start
```

Система аудита активируется и начнёт собирать события в соответствии с настройками по умолчанию.

### Остановка и просмотр логов

#### Остановка мониторинга

```cmd
auditd.exe --stop
```

#### Просмотр логов

```cmd
auditd.exe --show-logs
```

Логи по умолчанию сохраняются в:

```text
%ProgramData%\AuditD\logs\
```

### Настройка

Поведение AuditD можно изменить, создав файл `auditd.conf` в той же папке, где находится `auditd.exe`.

Пример базовой конфигурации:

```ini
[monitoring]
enable_process_events = true
enable_file_events = true
enable_registry_events = false

[storage]
log_dir = C:\CustomLogs
max_log_size_mb = 100
```

Подробное описание всех параметров смотрите в файле:

```text
docs/configuration.md
```

## ⚠️ Требования

* **Операционная система:** Windows 10 / Windows 11 / Windows Server 2016 и новее.
* **Права:** запуск от имени администратора (требуется для установки обработчиков событий и системного мониторинга).
* **Дисковое пространство:** около 5 МБ для исполняемого файла плюс место для логов (настраивается).

## ❓ Часто задаваемые вопросы

### Почему требуется запуск от имени администратора?

AuditD перехватывает системные вызовы и события на уровне ядра Windows, что требует повышенных привилегий. Без прав администратора утилита не сможет зарегистрировать обработчики событий.

### Антивирус сообщает о потенциальной угрозе. Это нормально?

Некоторые антивирусные движки могут ложно срабатывать на упакованные EXE-файлы (например, при использовании UPX для уменьшения размера) или на программы, перехватывающие системные события.

AuditD **не содержит вредоносного кода**. Вы можете проверить исходный код или собрать утилиту самостоятельно, чтобы исключить ложные срабатывания.

### Как просмотреть логи в реальном времени?

Используйте команду:

```cmd
auditd.exe --tail
```

Логи будут выводиться в консоль по мере поступления событий.

### Можно ли запускать AuditD как службу Windows?

Да, выполните:

```cmd
auditd.exe --install-service
```

После этого AuditD будет автоматически стартовать при загрузке системы и работать в фоновом режиме.

Для удаления службы используйте:

```cmd
auditd.exe --uninstall-service
```
